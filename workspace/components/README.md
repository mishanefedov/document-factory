# workspace/components

Your HTML components — atomic fragments an agent composes into a full doc.

- `headers/` — brand-specific top-of-doc mastheads (one per brand variant)
- `footers/` — brand-specific bottom-of-doc footers
- `blocks/` — reusable content blocks: stat-row, signal-box, blockquote, two-col-list, data-table, caveats

Each component file opens with a comment header that tells an agent what it does, when to use it, and what slots it has. See `instructions/component-conventions.md` for authoring rules.
