# workspace/examples

Your canonical shipped docs. The agent reads these to learn "what good looks like" for your brand.

## Curation rule

Add a doc to `examples/` when it represents your best execution — tight voice, correct structure, effective for the audience. Pair each example with a short note (either inline as an HTML comment at the top, or in a paired `{filename}.md`) describing what the doc teaches.

Examples:
- Good use of a Caveats block
- Strong quote-selection pattern
- Effective cross-respondent signal matrix
- Well-framed back-test section

## What not to put here

- Drafts that didn't ship (they go in `results/` or get deleted)
- Docs with content-level errors (fix before adding)
- Generic templates (those live in `templates/`)
